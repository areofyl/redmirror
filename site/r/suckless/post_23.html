st - How to disable automatic scrolldown on command output?

by Obertuba | 2 points

As the title says, I'm trying to figure out how to disable automatic scrolldown on command output, but keep automatic scrolldown on user input.

A possible use case for this would be reading through a command output that is still running without the need for piping into `less`.

What I tried (st-0.9.3):

From what I understand, the `twrite()` function is responsible for the automatic scrolldown, by setting `TSCREEN.off = 0`, whenever the terminal screen is written to. That means both on user input and on command output.

The `ttyread()` function is responsible for invoking `twrite()` on command output, and thus causing the automatic scrolldown behavior.

I tried passing a flag to twrite() to enable/disable resetting `TSCREEN.off` based on whether it's called from `ttyread()` or from another location. However, command output now overwrites the current screen content and I got a SEGFAULT after a while.

This makes me think that the automatic scrolldown on command output is quite entangled with other terminal functionality.

Has anyone succeeded in doing this?
