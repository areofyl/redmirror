dcpugalaxy 1

The basic idea is that some GUI programs are used to launch other GUI programs and usually you don't the launcher to stay around while you are using the launched program.

For example if you start a program from the terminal you usually don't need the terminal to stay visible while you use the program you launched. When you are done with the launched program you want the terminal to come back.

The swallow patch achieves this object by reusing the launcher's window for the new process.

So for example if you have a terminal open and you launch vim in it, vim will run in that terminal window (because it interacts with the tty that the terminal emulator presents.) But if you run gvim, it creates a new window.

If you use swallow then run gvim from your terminal, the terminal's window will be reused by gvim. Then when you close gvim the terminal emulator returns.
