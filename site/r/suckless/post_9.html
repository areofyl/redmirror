By default dwm will respect size hints for tiled windows. This will in particular affect windows that have aspect ratios like shown in the screenshots.

You can set resizehints to 0 in your config.h if you want to disable this feature - it may address some of the gaps but may not necessarily look better.

For the first screenshot it is not clear what layout you are using. The default tile layout with incremented nmaster would give a similar result, just that all windows would be top aligned.

I implemented window snapping for floating windows in my own build, but I think this wouldn't solve your problem as you'd want to use as much of the screen real-estate as possible (i.e. full tiling).

The various grid-based layouts available won't help here unless you disable resizehints.

cfacts may potentially help with some minor fine-tuning.

I suppose that you could have a bash script that uses e.g. wmctrl, xdo or xdotool to handle the sizing and positioning of floating windows, but this wouldn't reduce the complexity I feel.

Out of interest (and for fun) I did implement a grid layout that attempts to optimise the positioning and sizing of windows based on their aspect ratio, but this would be far into the "turning things into a mess" side for a minimalistic build.

Feel free to reach out via chat or otherwise if you'd like to discuss.
