wm (1) - A Static Brutalist Tiling WM for UNIX systems.

by LightPrototypeKiller | 61 points

DWM is a headache to work on, so I wrote my own tiling WM which has the following features and nothing more:

  • Mod + Return : Spawn terminal
  • Mod + Space : Spawn application menu
  • Mod + j / k : Focus next / previous window
  • Mod + j / k + Shift : Move focused window left / right
  • Mod + h / l : Shrink / expand the master area
  • Mod + f : Toggle fullscreen for the focused window
  • Mod + q : Kill the focused window
  • Mod + q + Shift : Quit wm
  • Mod + [1-9] : Switch to workspace N
  • Mod + [1-9] + Shift: Move focused window to workspace N

It's completely static using no dynamic memory allocation.

It uses OpenBSD's pledge() when it can.

It's around 250 lines of code.

It uses 2MB ram.

https://github.com/173duprot/wm
