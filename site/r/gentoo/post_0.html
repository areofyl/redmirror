TheShredder9 11

  1. No need to compile everything, the binhost is already preconfigured, you just need to tell Portage to use the precompiled binaries instead of compiling locally.

  2. As far as i know, for a while now the compilation flags are defaulted to -j $(nproc), which uses 100% of your CPU to compile, it used to not be the case and would use only 1 thread which could take many hours to compile anything

  3. Gentoolkit (i think that's the name) has a few goodies for packages such as equery, where you can see why a package is installed, what depends on it and many more things.

  4. Gentoo has "overlays", which are basically user repositories, the "official" one being GURU, i'd say that's the biggest AUR equivalent, though there are many of them.

  5. No memes, we are serious here ;)

Good luck!
