varsnef 1

Ok, you ran out of RAM in the VM and the kernel stopped the compilation.

You can give the VM more RAM or reduce the number of jobs the compilation uses.

cp1plus (c++) can easily use 1-2G of RAM per job. 3 or more with debug.

One option is to change the MAKEOPTS value only for firefox and reduce the number jobs that it will use during compilation.

Create a new file /etc/portage/package.env with the contents:

 www-client/firefox halfjobs.conf

And create a new directory /etc/portage/env

And a new file /etc/portage/env/halfjobs.conf with the contents:

MAKEOPTS="${MAKEOPTS} -j$(($(nproc)/2))

This will use half of the processor 'cores' that you gave the VM to compile firefox.

Half might be a bit too much, it will be slower. use MAKEOPTS=-j3 for only 3 cores or one or two lower than what you have.

4G of RAM you might be able to use -j3.
