especially

https://wiki.gentoo.org/wiki/Kernel/Gentoo_Kernel_Configuration_Guide#Enabling_support_for_typical_system_components and the topics after that....kernel panic with mount error often comes from lacking sata and/or filesystem driver.

An alternative is building the preconfigured gentoo kernel at first and once you have a somewhat working system go back and do your own kernel configuration. This way you have at least a fallback kernel until your get the kernel config right.

Edit:Oh..and there is almost never a need to "reinstall" gentoo...you usually can recover a system rather easily. In case of a wrong kernel config it's just booting the liveUSB again, chroot back in the gentoo system and redoing the kernel build step with corrected config this time.

Don't start from scratch. Chroot to your Gentoo and install gentoo-kernel-bin (read the handbook and choose the needed flags for grub/dracut/etc.) and the grub-mkconfig -o /boot/grub/grub.cfg 

It should find the gentoo-kernel-bin and then exit chroot, reboot, choose the new entry from Grub menu and you should have login prompt. 

Never start from scratch if you can chroot back!
