anh0516 6

Ok, so you've almost certainly got a bad kernel configuration for your hardware.

What you should do is boot back into the installation media, and then chroot and install gentoo-kernel-bin as directed: https://wiki.gentoo.org/wiki/Handbook:AMD64/Installation/Kernel#Installing_a_distribution_kernel

Once you've got your system up and running the way you want it, then you can start messing with the kernel configuration. You should copy the kernel configuration file from gentoo-kernel-bin and use that as a starting point for configuring your own kernel, rather than relying on the implicit defaults.

It takes a lot of work to go through every single menu option and configure a kernel that has only exactly what you need and nothing more. Expect to go through several recompilations.

NopeNotJayILeft 3

So a few suggestions: - Boot back into the liveusb, re-mount your root FS and chroot in with arch-chroot and you can resume your install/recover your machine. - Once in there, install gentoo-kernel-bin and linux-firmware to get you to a known-working spot. - Additionally, I recommend -- in the strongest terms -- using a full bootloader, like grub, which can help you troubleshoot broken kernels WHILE giving you a way to fallback to the previous kernel.

Once you have a fully working install with a distribution kernel, you can then look at things and see what might be missing from your custom one. What's the difference between /proc/config.gz (when zcat'd) when booted in that kernel vs the .config in the kernel dir you're using? Anything in lsmod you forgot to build as a module?

Building a custom kernel is work a lot of folks find satisfying, but it's 100% OK to use a distribution kernel (gentoo-kernel or gentoo-kernel-bin) long term -- that's what I do, mainly because I don't wanna have to go 10 rounds with make menuconfig anytime I update hardware or the kernel changes significantly :D.
