Sure the Gentoo Handbook is gonna be where you read about it, but its a huge ass tomb so lets assume you are more specifically asking which parts are more immediately useful to a beginner in a "pregame research" sort of way before you start blindly following the step by step guide.

First and foremost: Make, and GCC. You don't gotta learn C/C++, or get a deep understanding of compiler theory. You just want a good solid concept of using them as tools. Almost all other concepts in Gentoo spring from these two tools. Flags, buildopts, etc. "Make.conf" is the literal lynchpin of Gentoo. Even if you want to use an LLVM based system, still start with GCC at this "as a tool" level because LLVM will mostly emulate or implement the same concepts and options.

Second: Portage. It's more a broad concept than a specific tool. You will end up using different tools to do different things within the broader "Portage" system. "emerge" tends to be the main interface, but other tools like "eselect" or "equery" get plenty of use.

Building your own kernel is the next best thing to read up on, but that's more optional because there are binary or automated options available.
