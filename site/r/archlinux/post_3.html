Starting the investigtion with the system journal is always good:

journalctl -b -p 4
# Gives all errors and warnings on the system for the current boot.
journalctl -b -1
# For the previous boot. -b -2 for second previous, etc.
journalctl | grep -i niri
# You can search for keywords in the journal. -i for case-insensitive search.

Otherwise you can check how you can get the app-specific logs for niri, ghostty, nicotine+.

And your GPU might be related too. You can add information about it to your post, from lspci -k.

Also check for any recent issues on niri's github issue tracker. Perhaps there is some insight there. For example, could this issue be related?

Good luck!
