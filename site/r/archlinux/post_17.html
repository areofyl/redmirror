sausix 2

Run efibootmgr to see a list of the registered bootloaders and order. Maybe you have a wrong one set for booting up. Is /boot/efi your mounted UEFI partition?

And maybe this is your moment of realizing grub isn't always easy and stable and you may switch to a more modern bootloader. With your Arch Linux powers (and the wiki) you are free to choose from multiples ones instead of having grub preinstalled on beginner distros. While some people claim they "never" had any problems other people often end up in the grub-rescue environment and ask for help in the communities.
If you don't have any good reason for using grub then try rEFInd for example. It's not bound to a specific OS and has dynamic but statically configurable logic to detect OSes and kernels. It's not bound to any dynamic config generation which often is the breaking point for grub.

I checked using gparted and the efi partition is mounted at /boot, so I reinstalled grub at /boot instead and now it somewhat works. The entries for it are now: Arch with zen, Arch with surface, Arch with lts, Arch linux, Arch linux, Windows 10, Uefi settings, Windows 10. The first and second "Arch linux" still refer to zen and lts but now there are also labeled entries for each of the 3 kernels and windows 10 shows up twice for some reason

Not ideal but this is usable for now since my main goal was using linux-surface so thanks for the help. I'll mark this as solved but I'm curious about the other modern bootloaders you mentioned that are easier to configure and don't break so easily. Could be useful to know for next time

Edit: The duplicate entries were caused by having 2 os-prober scripts and a uki script in /etc/grub.d. Deleting the second os-prober script removed the duplicate windows entry and deleting the uki script removed the non-descriptive "Arch Linux" entries. Now it's back to how it was before with just the 5 essential entries
