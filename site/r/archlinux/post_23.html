Podman: do I need AppArmor if I use krun as a runtime?

by Wise_Stick9613 | 2 points

AppArmor is listed as an optional dependency for Podman and I chose krun as the runtime.

According to ArchWiki, krun is:

a special runtime that uses KVM-backed krunvm-based micro VMs to execute the container. Compared to a full VM, these micro VMs start in milliseconds and use a different kernel. This should provide better security compared to regular containers that run with the host kernel.

Since we're talking about VMs, does it still make sense to install AppArmor, or is it pointless?

krun with those micro VMs basically gives you kernel-level isolation already so AppArmor becomes pretty redundant in that setup. The whole point of AppArmor is controlling what processes can access on host kernel but when you're running in separate VM kernel that protection layer isn't really needed anymore

I'd skip it unless you plan on mixing runtimes or running some containers with regular runc
