The trick macOS does to accomplish this is currently not implemented. If you're fine having to always attach the external drive to boot into Asahi, you could just btrfs device add it to your main filesystem. Then your Fedora Asahi will effectively be installed on both the internal drive and the external one. This way you can get away with only assigning Asahi the bare minimum of internal storage that's necessary for install, and still have a lot of space for any later needs.

Another easy way is using the external drive to store only certain categories of files. For example, you can have the OS installed onto the internal drive, but store your entire home directory (~) on the external one.

And if you're proficient with Linux administration, it is perfectly possible to move the entire OS partition to the external storage, only leaving the bootloader on the main drive.

It’s impossible to fully boot from external media. The firmware deliberately doesn’t support it. In future IIRC it may be possible to install with only m1n1 stage 1 (not sure if stage 2 is required) on the internal drive, without repartitioning. But that’s not possible rn. The best you can do is to have /boot (and the Asahi-specific partitions) on the internal drive. However, that isn’t plug and play. It requires at least a basic understanding of how to configure disks on Linux.
