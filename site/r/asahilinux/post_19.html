Today I have update to Fedora 44 Beta and it's okay with few issues

by fake_agent_smith | 22 points

So today I've used the standard dnf system plugin, the update itself went well but a word of caution: I as many other people here use the custom build based on fairydust branch and probably because of that during update to F44 the process didn't create initramfs for the standard asahi kernel.

After boot I've noticed that DP Alt mode no longer works and I quickly realized that the typec kernel modules are not loaded. For whatever reason on F43 they loaded automatically(?) and on F44 I had to create a custom modules-load.d file:

cat /etc/modules-load.d/fairydust.conf
typec_displayport
typec_nvidia
typec_thunderbolt

Another issue I immediately noticed is that when logging in, the login sound is super loud. I'm not sure why but after login if I don't modify volume it's stuck at probably 100%. speakersafetyd works correctly, tried reinstalling both it and asahi-audio, tried to remove wireplumber state because I thought maybe it's stale, but that didn't change anything so, for now I have simply disabled the login sound and create a user systemd unit as a temporary fix to make sure I don't overstretch my laptops speakers:

[Unit]
Description=Fix WirePlumber Volue at Login
After=wireplumber.service
Requires=wireplumber.service

[Service]
Type=oneshot
# We wait 2 seconds to ensure the audio stack is fully awake
ExecStartPre=/bin/sleep 2
ExecStart=/usr/bin/wpctl set-volume @DEFAULT_AUDIO_SINK@ 1%-
ExecStartPost=/usr/bin/wpctl set-volume @DEFAULT_AUDIO_SINK@ 1%+

[Install]
WantedBy=default.target

Other than I didn't notice any breakage in user space.
